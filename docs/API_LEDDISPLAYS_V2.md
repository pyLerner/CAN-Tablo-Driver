# REST API v2 — справочник эндпоинтов (внешние сервисы)

Краткое описание HTTP-интерфейса для интеграции **без** внутренней логики сервера. **Один экземпляр сервиса — одно табло** (одна пара CAN ID в `[display]`). Базовый URL задаётся при развёртывании, например `http://<хост>:<порт>` (хост и порт — секция `[api-server]` в `config.toml`).

Префикс путей остаётся `/api/leddisplays/v1/` для совместимости с существующими клиентами.

## Соглашения

| Параметр | Значение |
|----------|----------|
| Заголовок запросов с телом | `Content-Type: application/json` |
| Формат ответов | JSON, UTF-8 |
| Ключи JSON | **kebab-case** (`first-string`, `route-width`, …) |
| Версия пути | префикс `/api/leddisplays/v1/` (кроме ping) |

### Предварительные соглашения по ID зон

| ID зоны | Назначение |
|:-------:|------------|
| `1` | Cтрока с номером маршрута |
| `2` | Cтрока с начальной остановкой |
| `3` | Cтрока с конечной остановкой |
| `4` | Cтрока со следующими остановками (выборочно) |
| `8` | Cтрока для бегущей строки в салоне |
| `9` | Cтрока для бегущей строки в салоне (опциональная) |

## `GET /api/ping`

Проверка доступности сервиса.

**Ответ `200 OK`**

| Поле | Тип | Описание |
|------|-----|----------|
| `running` | string | При успехе: `"OK"` |
| `timestamp-utc` | string | Время в UTC (ISO 8601, суффикс `+00:00`) |
| `display-id` | string | Идентификатор табло (для отладки и логов) |

```json
{
  "running": "OK",
  "timestamp-utc": "2026-03-29T12:34:56.789+00:00",
  "display-id": "front-display"
}
```

## `POST /api/leddisplays/v1/config/set`

Обновление параметров табло и общих настроек. Отсутствующие в теле поля **не сбрасывают** ранее сохранённые значения.

**Тело запроса** — все поля опциональны; указываются только изменяемые блоки.

| Поле | Тип | Описание |
|------|-----|----------|
| `color-map` | dictionary&lt;string, Color&gt; | Палитра цветов для зон (индексы `"0"`–`"15"`). Значение — объект RGB |
| `zones` | dictionary&lt;string, Zone&gt; | Настройки зон табло (индексы `"1"`–`"10"`). Каждая зона — строка текста на дисплее |

**Color** — элемент `color-map`

| Поле | Тип | Описание |
|------|-----|----------|
| `r` | int | Красный канал (0–255) |
| `g` | int | Зелёный канал (0–255) |
| `b` | int | Синий канал (0–255) |

**Zone** — элемент `zones`

| Поле | Тип | Описание |
|------|-----|----------|
| `bg` | int | Цвет фона (0–15, индекс из `color-map`) |
| `fg` | int | Цвет текста (0–15, индекс из `color-map`) |
| `area` | Area | Координаты и размеры зоны в точках от верхнего левого угла табло |
| `padding` | Padding | Отступы текста внутри зоны в точках |
| `font` | int | Индекс шрифта |

**Area** — координаты зоны

| Поле | Тип | Описание |
|------|-----|----------|
| `x` | int | Горизонтальное смещение (от левого края, в точках) |
| `y` | int | Вертикальное смещение (от верхнего края, в точках) |
| `w` | int | Ширина зоны (в точках) |
| `h` | int | Высота зоны (в точках) |

**Padding** — отступы внутри зоны

| Поле | Тип | Описание |
|------|-----|----------|
| `t` | int | Отступ сверху (в точках) |
| `r` | int | Отступ справа (в точках) |
| `b` | int | Отступ снизу (в точках) |
| `l` | int | Отступ слева (в точках) |

```json
{
    "color-map":{ 

        "0": {
            "r": 0,
            "g": 0,
            "b": 0
        }, 
        "1":{
            "r": 255,
            "g": 255,
            "b": 255
        },
        "2":{
            "r": 255,
            "g": 255,
            "b": 0
        },
        ...
    },
    "zones": { 
        "1": { 
            "bg": 2, 
            "fg": 0, 
            "font": 4,
            "area": { 
                "x": 0,
                "y": 0,
                "w": 100,
                "h": 50
            },
            "padding": { 
                "t": 1,
                "r": 2,
                "b": 1,
                "l": 2
            }
        },
        "2": {...},    
        "3": {...},    
        ...
    
    }
}
```

| Код | Тело (пример) | Когда |
|-----|----------------|--------|
| `200` | `{"status":"ok"}` | Изменения применены |
| `200` | `{"status":"noop"}` | Пустое тело обновлений (нечего менять) |
| `422` | ошибки валидации | Неверная структура или типы |

## `PUT /api/leddisplays/v1/values/update`

Задать текст для зон табло и инициировать **асинхронную** отправку на **единственное** табло, заданное в конфигурации (`[display]`). Отображаются только зоны, описанные в конфиге (`[display.N]`); ключи в `values` сопоставляются с номерами зон.

**Тело запроса**

| Поле | Тип | Описание |
|------|-----|----------|
| `values` | dictionary<int,string> | текст для зон по индексам |

```json
{
  "values": {  
      "1": "567А",
      "2": "Начальная остановка",
      "3": "Конечная остановка",
      "8": "Текст бегущей строки",
      "9": "Текст бегущей строки"
  }
}
```

**Ответ `200 OK`**

```json
{ "status": "accepted" }
```

Отправка на оборудование выполняется асинхронно после ответа.

| Код | Когда |
|-----|--------|
| `422` | Ошибка валидации тела |

## Поведение при рендеринге (сервер)

- Если **сумма** полей `area.h` по всем зонам в конфиге **больше** `display.height`, в лог пишется сообщение об **ошибке** (отправка при этом всё равно выполняется).
- Если две зоны имеют пересечение прямоугольников `area` **ненулевой площади**, в лог пишется **ошибка** с указанием id зон.
- Если правый край разметки выходит за `display.width` (`max(x+w) > width`), для зон с **максимальным** `area.x` принудительно используется режим бегущей строки (opcode `0x0002`).
- Палитра `color-map`: на шину передаётся один байт цвета; в полной мере учитываются слоты, соответствующие **чёрному** и **жёлтому** RGB; прочие индексы в этой версии на проводе приводятся к правилам по умолчанию (fg → жёлтый, bg → чёрный).
- Индекс шрифта **1** сопоставлен основному TTF (см. `[TextIn].font` и опционально `[fonts]` в TOML); остальные индексы в этой версии не используются.
