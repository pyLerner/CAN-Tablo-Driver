# REST API v1 — справочник эндпоинтов (внешние сервисы)

> **Устарело.** Используйте **[API_LEDDISPLAYS_V2.md](API_LEDDISPLAYS_V2.md)**. Этот файл описывает удалённую модель нескольких табло и эндпоинты `route/set`, `tickerboard/set`.

Краткое описание HTTP-интерфейса для интеграции **без** внутренней логики сервера. Базовый URL задаётся при развёртывании, например `http://<хост>:<порт>` (хост и порт настраиваются в `[api-server]` в конфигурации сервиса).

## Соглашения

| Параметр | Значение |
|----------|----------|
| Заголовок запросов с телом | `Content-Type: application/json` |
| Формат ответов | JSON, UTF-8 |
| Ключи JSON | **kebab-case** (`first-string`, `route-width`, …) |
| Версия пути | префикс `/api/leddisplays/v1/` (кроме ping) |

---

## `GET /api/ping`

Проверка доступности сервиса.

**Ответ `200 OK`**

| Поле | Тип | Описание |
|------|-----|----------|
| `running` | string | При успехе: `"OK"` |
| `timestamp-utc` | string | Время в UTC (ISO 8601, суффикс `+00:00`) |

```json
{
  "running": "OK",
  "timestamp-utc": "2026-03-29T12:34:56.789+00:00"
}
```

---

## `POST /api/leddisplays/v1/config/set`

Частичное обновление параметров табло и общих настроек. Отсутствующие в теле поля **не сбрасывают** ранее сохранённые значения.

**Тело запроса** — все поля опциональны; указываются только изменяемые блоки.

| Поле верхнего уровня | Тип | Описание |
|---------------------|-----|----------|
| `color` | string | Цвет текста, например `"YELLOW"`, `"RED"` (см. допустимые имена на стороне сервиса) |
| `channel` | string | Интерфейс CAN, например `can0` |
| `bitrate` | integer | Скорость шины, бит/с |
| `front-display` | object | Параметры переднего табло |
| `side-front-display` | object | Боковое (передняя сторона) |
| `side-rear-display` | object | Боковое (задняя сторона) |
| `rear-display` | object | Задний маршрутный указатель |
| `ticker-board` | object или array of object | Одно или несколько табло бегущей строки |

Во вложенных объектах табло (по необходимости):

| Поле | Тип | Описание |
|------|-----|----------|
| `sender-tx-id` | integer | CAN ID передачи |
| `sender-rx-id` | integer | CAN ID приёма |
| `route-width` | integer | Ширина колонки маршрута |
| `route-text-scale-x` | number | Масштаб текста маршрута по горизонтали |
| `width`, `height` | integer | Размер области |
| `pad-left`, `pad-right`, `pad-top`, `pad-bottom` | integer | Отступы |
| `right-lines` | integer | `1` или `2` — строк справа от маршрута (маршрутные табло) |
| `ticker-lines` | integer | `1` или `2` — режим строк тикера |

**Ответы**

| Код | Тело (пример) | Когда |
|-----|----------------|--------|
| `200` | `{"status":"ok"}` | Изменения применены |
| `200` | `{"status":"noop"}` | Нечего обновлять (пустое тело после разбора) |
| `400` | `detail` с текстом ошибки | Недопустимое значение, напр. `color` |
| `422` | ошибки валидации | Неверная структура или типы |

## `POST /api/leddisplays/v1/route/set`

Задать три строки маршрута для отображения и инициировать отправку на все настроенные **маршрутные** табло (переднее, боковые, заднее). Состояние тикера этим запросом не меняется.

**Тело запроса**

| Поле | Тип | Описание |
|------|-----|----------|
| `first-string` | string | Номер маршрута / основная зона (аналог `firstString` во внутреннем JSON) |
| `second-string` | string | Первая строка справа от номера |
| `third-string` | string | Вторая строка справа от номера |

```json
{
  "first-string": "567А",
  "second-string": "Начальная остановка",
  "third-string": "Конечная остановка"
}
```

**Ответ `200 OK`**

```json
{ "status": "accepted" }
```

Отправка на оборудование выполняется асинхронно после ответа.

| Код | Когда |
|-----|--------|
| `422` | Ошибка валидации тела |

---

## `GET /api/leddisplays/v1/route/set`

Получить последние сохранённые сервисом три строки маршрута (после успешного `POST` или пустые строки до первого задания).

**Ответ `200 OK`**

| Поле | Тип |
|------|-----|
| `first-string` | string |
| `second-string` | string |
| `third-string` | string |

```json
{
  "first-string": "567А",
  "second-string": "Начальная остановка",
  "third-string": "Конечная остановка"
}
```

Тот же путь и ресурс, что у `POST`, отличается только метод HTTP.

---

## `POST /api/leddisplays/v1/tickerboard/set`

Задать две строки внутрисалонного тикера и инициировать отправку **только** на все настроенные табло `ticker-board`. Маршрутные табло не затрагиваются.

**Тело запроса**

| Поле | Тип | Описание |
|------|-----|----------|
| `first-string` | string | Первая строка / верхний блок |
| `second-string` | string | Вторая строка / нижний блок |

```json
{
  "first-string": "Текст 1",
  "second-string": "Текст 2"
}
```

**Ответ `200 OK`**

```json
{ "status": "accepted" }
```

| Код | Когда |
|-----|--------|
| `422` | Ошибка валидации |
